Vue d'ensemble & Guide d'Installation

Bienvenue sur la documentation technique des ressources de la suite Yol_Script.

Cette suite de scripts a été conçue pour fonctionner de manière optimisée sous le framework ESX Legacy, en utilisant les briques les plus performantes de l'écosystème FiveM actuel : la base de données oxmysql, la bibliothèque utilitaire ox_lib, l'inventaire ox_inventory, et le script d'interactions physiques ox_target.

Design System Unifié Toutes les interfaces de la suite Yol partagent le même design premium « Glassmorphism » basé sur la couleur d'accentuation orange #F04C00. Le style est centralisé dans le script Yol_lib, évitant ainsi le clonage du code CSS.

Le serveur n'exécute aucun script externe non configuré, et tout le code est articulé pour maximiser le réalisme, le confort de jeu et la sécurité (anti-triche de haut niveau intégré).

Qu'est-ce que l'écosystème Yol_Edition ?

La suite Yol_Edition est un écosystème modulaire clé-en-main conçu pour optimiser, administrer et animer votre serveur FiveM. Géré par un dashboard NUI en jeu et synchronisé avec un bot Discord natif, il intègre des fonctionnalités d'administration, de sécurité, de communication moderne et de gameplay enrichi.

🤖 Bot Discord NodeJS Natif

La suite Yol_Edition intègre un bot Discord programmé en NodeJS sous la bibliothèque discord.js, situé dans le dossier 📁 Yol_Admin/bot/.

Étapes d'installation du Bot :

  1. Ouvrez un terminal de commande dans le dossier 📁 Yol_Admin/bot/.
  2. Exécutez la commande suivante pour installer les paquets Node.js requis : 
    npm install  # ou yarn install
  3. Renommez le fichier config.example.json en config.json.
  4. Éditez le fichier config.json en renseignant vos identifiants réels : 
    {
  "token": "BOT_DISCORD_TOKEN_SECRET",
  "guildId": "ID_DE_VOTRE_SERVEUR_DISCORD",
  "roleIds": {
    "admin": "ID_ROLE_ADMINISTRATEUR",
    "mod": "ID_ROLE_MODERATEUR"
  },
  "channelIds": {
    "reports": "ID_SALON_SIGNALEMENTS",
    "logs": "ID_SALON_JOURNAL_LOGS"
  }
}
  5. Lancez le bot en tâche de fond à l'aide d'un gestionnaire de processus comme PM2 : 
    pm2 start index.js --name "yol-bot"

Dépendances & Écosystème (OX, mVehicle, pefcl...)

Chaque script Yol interagit étroitement avec des briques techniques spécialisées. Voici une explication approfondie de ces dépendances requises présentes dans votre dossier de ressources :

1. La Suite Overextended (OX)

  • oxmysql : Pilote BDD asynchrone ultra-rapide. Il gère l'ensemble des requêtes SQL de la suite (Yol_Phone, Yol_EMS, Yol_Admin, etc.).
  • ox_lib : Expose des composants cruciaux (callbacks serveur-client sécurisés, barres de progression, menus contextuels, et utilitaires de thread).
  • ox_inventory : Système d'inventaire à base d'items physiques dotés de métadonnées uniques. Indispensable pour lier le téléphone au fait de posséder l'item phone (portant l'identifiant unique SIM), stocker les permis Yol_Licenses avec photo et identité, et gérer les stashes de vêtements Yol_Clothing.
  • ox_target : Système de ciblage 3D qui élimine le spam de marqueurs 3D au sol et optimise la consommation CPU client (0.00ms au repos).
  • ox_doorlock : Gestionnaire de verrous de portes physiques de la map, synchronisé en direct avec le panel de création visuelle de Yol_Admin.

2. L'écosystème Véhicules (mVehicle & mGarage)

  • mVehicle (Mono-94) : Framework de persistance des véhicules. Il gère l'état physique complet des voitures (moteur, carrosserie, pneus, essence), l'attribution et le partage des clés de contact et la génération des plaques d'immatriculation.
  • mGarage (Mono-94) : Système de stationnement et de fourrière. Il permet de stocker les véhicules de façon persistante en BDD et d'en retirer les clés. Il s'intègre avec l'application Garage de Yol_Phone (permettant la localisation GPS, le verrouillage/déverrouillage à distance, et l'allumage des moteurs/phares depuis le téléphone).

3. Le Système Bancaire (pefcl)

  • pefcl (Aura Bank) : Système financier avancé. **Il sert de moteur bancaire principal universel pour l'ensemble des scripts de la suite Yol**. Absolument toutes les transactions financières du serveur (versements des salaires, facturation des interventions médicales Yol_EMS, achat de permis Yol_Licenses, contraventions émises par Yol_Police, application Banque de Yol_Phone ou frais d'utilisation de Yol_Cruisecontrol) transitent par les exports de pefcl. C'est l'épine dorsale de l'économie.

4. La Gestion de la Météo (B2_WeatherEssentials)

  • B2_WeatherEssentials (b2_weatherEssentials) : Gestionnaire d'environnement unifié. Il synchronise l'heure, l'eau, et les météos régionales de manière autoritaire sur le serveur. Il intègre des saisons (printemps, été, automne, hiver), un système de vote des joueurs, un mode Blackout (panne électrique), et des événements extrêmes (tsunami, tremblement de terre, ouragan, pluie de météores). Ses commandes administratives et NUI sont entièrement sécurisées via ACE.

Base de données & Schémas SQL

La persistance de vos scripts repose sur la base de données MySQL. Plusieurs scripts Yol intègrent un système d'auto-setup qui crée automatiquement leurs tables au premier boot (Yol_Phone, Yol_EMS, Yol_Licenses via MySQL.ready). D'autres nécessitent une vérification de colonnes.

Auto-Migration Active Les scripts Yol vérifient et adaptent automatiquement le format des colonnes (comme la modification de yol_phone_photos.url en LONGTEXT pour stocker les images base64 locales).

Table : yol_licenses

Stocke les informations des cartes d'identité, permis de conduire et permis de port d'arme physiques générés par Yol_Licenses.

Colonne Type de Donnée Index / Clé Description
id INT PK A_I Identifiant unique de la licence en BDD.
license VARCHAR(128) INDEX Identifiant de licence Cfx.re du propriétaire.
type VARCHAR(64) — Type de permis (ex: drive, weapon, identity).
number VARCHAR(64) UNIQUE Numéro de permis unique généré aléatoirement.
data JSON — Métadonnées (nom, prénom, photo_url issue de Discord Webhook, date de naissance).
created_at TIMESTAMP — Date de création de la carte.

Table : user_injuries

Permissions ACE (perms.cfg)

Le serveur utilise le système de permissions natif de GTA V/FiveM appelé ACE. Ce système définit des règles d'accès strictes pour les commandes et les ressources en jeu (sans surcharger la base de données).

Voici l'intégralité du fichier de permissions perms.cfg configuré pour Yol_Script. Vous pouvez copier ce bloc complet dans votre configuration :

perms.cfg 
# # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # #
#                       CONFIGURATIONS DES PERMISSIONS                        #
# # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # # #

# ==========================================
#  Administrateurs Système (Principals)
# ==========================================
add_principal identifier.fivem:4274194 group.admin # EddyFinch
add_principal identifier.discord:150485416788426753 group.admin # Admin Principal

# ==========================================
#  Hiérarchie des Groupes
# ==========================================
add_principal group.admin group.user

# ==========================================
#  Permissions ACE - Groupe Admin
# ==========================================
add_ace group.admin command allow
add_ace group.admin command.quit deny

# Commandes de gestion de l'environnement
add_ace group.admin command.setweather allow
add_ace group.admin command.settime allow

# Commandes de l'anti-triche Yol_Anticheat
add_ace group.admin command.acban allow
add_ace group.admin command.aclog allow
add_ace group.admin command.actest allow
add_ace group.admin command.acreload allow
add_ace group.admin command.actestmode allow

# ==========================================
#  Permissions ACE - Ressources & Framework
# ==========================================

# Permissions ESX Framework (es_extended)
add_ace resource.es_extended command.add_ace allow
add_ace resource.es_extended command.add_principal allow
add_ace resource.es_extended command.remove_principal allow
add_ace resource.es_extended command.stop allow

# Permissions Overextended Library (ox_lib)
add_ace resource.ox_lib command.add_ace allow
add_ace resource.ox_lib command.remove_ace allow
add_ace resource.ox_lib command.add_principal allow
add_ace resource.ox_lib command.remove_principal allow

# Permissions pour Yol_Admin
add_ace resource.Yol_Admin command.ensure allow
add_ace resource.Yol_Admin command.stop allow
add_ace resource.Yol_Admin command.start allow

# Permissions pour l'anti-triche et son Watchdog
add_ace resource.Yol_Anticheat_Watchdog command.ensure allow
add_ace resource.Yol_Anticheat command.ensure allow
add_ace resource.Yol_Anticheat command.add_ace allow

Explication détaillée des Règles ACE

  • add_principal [PlayerID] group.admin : Associe l'identifiant FiveM ou Discord d'un staff directement au groupe de modération group.admin.
  • add_ace group.admin command allow : Permet au groupe admin d'exécuter toutes les commandes natives ou scriptées du serveur.
  • add_ace group.admin command.quit deny : Empêche explicitement les admins d'exécuter la commande quit (qui couperait le serveur FiveM entier).
  • Permissions de Ressources (resource.XXX) :
    FiveM isole les scripts. Pour que Yol_Admin puisse lancer/arrêter ou redémarrer d'autres ressources depuis l'interface d'administration en jeu (onglet Scripts), la ressource Yol_Admin doit elle-même recevoir la permission d'exécuter les commandes console ensure, stop et start. De même, Yol_Anticheat_Watchdog a besoin de command.ensure pour relancer automatiquement l'anti-triche s'il est arrêté par un tricheur.

Configuration Globale (Convars)

De nombreux scripts de la suite Yol chargent leurs paramètres les plus sensibles (clés d'API, jetons de bots Discord, et Webhooks) directement depuis la configuration générale de la machine via des Convars. Cela évite de laisser ces identifiants visibles en clair dans les fichiers de scripts et permet de tout centraliser.

Ces variables sont définies dans le fichier config.cfg, qui est chargé au démarrage du serveur par le fichier principal server.cfg.

Sécurité des Webhooks & Tokens Ne partagez jamais vos fichiers server.cfg ou config.cfg. Les webhooks Discord permettent à quiconque les possède d'écrire sur vos salons, et vos jetons de bots peuvent compromettre votre serveur Discord.

Ordre de Démarrage des Ressources

Le chargement des scripts FiveM doit suivre un ordre strict pour éviter des erreurs de variables nulles (nil pointer) ou de callbacks introuvables au démarrage. Les dépendances de base (base de données, UI partagée) doivent être actives avant les scripts qui les utilisent.

L'ordre est géré de manière modulaire via le fichier resources.cfg.

resources.cfg 
# 1. Ressources Système de Base FiveM
ensure [BaseFivem]

# 2. Pilote BDD (oxmysql en premier)
ensure oxmysql

# 3. Anti-Triche et Watchdog (démarrés très tôt)
ensure Yol_Anticheat
ensure Yol_Anticheat_Watchdog

# 4. Librairies partagées & UI
ensure ox_lib
ensure Yol_lib
ensure Yol_Debugger

# 5. Ressources Noyau (Core)
ensure [Core]
ensure PolyZone
ensure ox_target
ensure Yol_Clothing
ensure ox_inventory

# 6. Dépendances de scripts additionnels
ensure [Depend]  # (Contient Yol_Bridge, bl_bridge, mVehicle, pma-voice, xsound, etc.)
ensure ox_doorlock

# 7. Framework ESX
ensure [ESX]

# 8. Ressources et scripts de vie / métiers
ensure [Others]  # (Contient Yol_Phone, Yol_EMS, Yol_Police, Yol_Core, Yol_Tchat, Yol_Licenses, etc.)
ensure Yol_Admin

Yol_Bridge

Dépendance Noyau Dossier: [Depend]/Yol_Bridge

Yol_Bridge est la couche d'abstraction centrale pour toutes les ressources développées par Yol. Elle se charge d'unifier les appels au framework. Bien qu'elle comporte du code pour d'autres environnements, elle est ici configurée pour lier entièrement vos scripts à ESX Legacy.

Dépendances

  • es_extended (framework hôte)

Fonctionnalités

  • Expose des exports serveur et client pour manipuler l'argent, les jobs, les permissions, l'identité et les notifications du joueur.
  • Fournit un fallback "standalone" propre si aucun framework n'est détecté.
  • Démarrage découplé : un polling de 30 secondes s'assure de lier le framework même si ESX démarre après Yol_Bridge.

Configuration

Le fichier de configuration principal est esx_shim.lua et la détection s'effectue dans bridge.lua.

Yol_lib

Dépendance Noyau Dossier: [Depend]/Yol_lib

Bibliothèque de composants UI partagée, extraite de Yol_Admin. Elle regroupe les feuilles de style CSS, les polices (Inter, Montserrat, FontAwesome) et les scripts JavaScript permettant d'afficher des notifications premium stylisées, des boîtes de confirmation et des fenêtres modales avec le thème Yol_Script Orange (#F04C00).

Dépendances

  • ox_lib

Fonctionnalités & Raccourcis

  • Système de notification riche : Titre, message, icône, barre de progression et glassmorphism.
  • Confirmations & Modals dynamiques en Javascript asynchrone (Promises).
  • Taper la commande /yollib_demo ouvre un catalogue complet de tous les éléments esthétiques (Showcase NUI).
  • Taper /yollib_test_notify [message] pour envoyer une notification de test rapide.

Fichiers de Configuration

  • shared/theme.lua : Déclare la palette de couleurs (YolTheme) répliquée côté client et serveur.
  • web/yol-lib.css : Contient les 900 lignes de définitions de design system (tokens CSS).
  • web/yol-lib.js : Fonctions d'API globales côté client web (window.YolLib).

Yol_Admin

Système & Modération Dossier: Yol_Admin

Le panneau d'administration tout-en-un de Yol_Script. C'est le centre de contrôle total du serveur, regroupant 24 modules et panels visuels interactifs ainsi qu'un bot Discord Node.js autonome.

Dépendances

  • es_extended
  • ox_lib
  • oxmysql
  • Yol_Bridge
  • ox_inventory (recommandé pour la fouille)
  • ox_target (recommandé pour les intéractions physiques)

Détail de ce que fait Yol_Admin (Par Panel NUI)

1. Modération & Gestion des Joueurs

  • Dashboard : Fournit un flux de statut (joueurs, ping moyen, statut des scripts BDD/Tebex) et intègre une mini-carte traquant en temps réel la position des joueurs.
  • Players : Liste exhaustive des joueurs (Online/Offline). Permet de Kick, Bannir (txAdmin ou local), Warn, Jail, Slap, donner un Job/Grade, modifier l'argent, Revive, Heal, se téléporter (Goto/Bring), Spectate en caméra libre ou activer les ESP/Noms/Blips.
  • Multi-PIP (Picture-in-Picture) : Outil de surveillance permettant d'ouvrir jusqu'à 4 fenêtres de spectate simultanées sur le même écran pour surveiller les scènes en cours.

2. Économie & Flotte de Véhicules

  • Eco : Statistiques macro-économiques du serveur (somme du cash global, banque, comptes de sociétés de factions). Met à jour les graphes toutes les 30 minutes.
  • Fleet & Vehicles : Liste et téléporte les véhicules actifs du serveur. Permet de supprimer (DV), réparer, laver, déverrouiller, inspecter le propriétaire d'une plaque d'immatriculation et gérer les voitures sauvegardées en BDD.

3. Éditeurs Visuels (Créateurs d'environnements)

  • Job & Gang Creator : Outil d'édition visuel pour ajouter, modifier ou renommer des grades, droits et salaires des factions ESX sans toucher aux fichiers SQL ou config.
  • Drug Creator : Place des points de récolte/traitement/revente sur la carte, lie des minijeux d'adresse, configure les gains d'argent sale et le taux d'appel de la police.
  • Zones Creator : Trace des zones géographiques (circulaires ou polygonales) et leur assigne des règles (Safezone invincible, zone sans armes, zone sans véhicule, limitation de vitesse).
  • Doorlock & Props Manager : Permet de configurer des portes ox_doorlock ou de spawner et placer des props de décoration persistants mondiaux via un outil laser de raycast 3D.
  • Inventory Creator : Permet de concevoir de nouveaux items de jeu en choisissant leur libellé, poids, description et image d'inventaire directement dans le NUI.

4. Gestion Serveur & Bot Discord

  • Server : Active la maintenance, la Whitelist par ID, édite les messages d'accueil et planifie des reboots serveur automatisés.
  • Console & Scripts : Stream le terminal du serveur en temps réel, exécute des commandes console, et liste toutes les ressources en permettant de les Démarrer, Stopper ou Redémarrer d'un clic.
  • Discord Bot : Démarre un bot Node.js lié à votre Discord depuis l'UI en jeu. Permet aux modérateurs d'exécuter des slash commands sur Discord (/joueurs, /say, /kick, /ban, /revive, /setjob) et d'envoyer des journaux log complets.

Fichiers de Configuration

  • shared/config.lua : Définit la langue, les groupes ACE administratifs autorisés, le mode de détection des dépendances et les hooks d'anti-triche.
  • shared/permissions_catalog.lua : Catalogue des permissions modifiables depuis le panel Permissions.
  • bot/config.json : Enregistre le token Discord, l'ID de la guilde, le rôle admin autorisé et le salon de logs (géré automatiquement par le panel Discord Bot).
  • data/robbery_config.json : Persiste la configuration des braquages configurés via l'UI Robbery Manager.

Commandes & Raccourcis

  • F6 (ou BACKSLASH) : Ouvre le menu d'administration principal.
  • INSERT : Ouvre le Quick Menu (options rapides joueur/véhicule).
  • F7 : Ouvre le panel console en jeu.
  • F9 : Active/Désactive le Noclip.
  • F10 : Active/Désactiver le Godmode.

Yol_Anticheat

Sécurité Dossier: Yol_Anticheat

Système anti-triche serveur rigoureux avec scan d'injection, détection de tricheurs en temps réel (vitesse, noclip, godmode, blacklist d'entités, anti-triggers) et création dynamique d'une Whitelist d'événements réseau sécurisée.

Dépendances

  • es_extended
  • oxmysql
  • screenshot-basic
  • Yol_Anticheat_Watchdog

Détails des Protections

  • Anti-Explosions & Anti-PNJ/Objets : Blocage et suppression instantanée des spams de cages, météores, géants ou d'explosions répétitives.
  • Anti-Triggers de Triche : Blocage direct de l'appel aux triggers sensibles de modération ou de framework (ex : esx:addInventoryItem initié par le client).
  • Event Whitelist Automatique : Analyse statique au démarrage de tous les scripts pour autoriser uniquement les requêtes légitimes. Enregistre les dérives en mode "LEARN" (durant 1h après reboot) puis bloque/bannit en mode "ENFORCE".

Fichiers de Configuration

  • config.lua : Déclare les listes d'armes et véhicules bannis, l'intervalle d'analyse des joueurs, le type de bannissement (txAdmin/BDD/Drop), et la liste noire des événements réseau (Naruto/Susanoo spams).

Yol_Anticheat_Watchdog

Sécurité Dossier: Yol_Anticheat_Watchdog

Script de sécurité indépendant agissant comme un "chien de garde". Son seul but est de s'assurer que l'anti-triche Yol_Anticheat est toujours en cours d'exécution. Si un tricheur de niveau administrateur tente de stopper la ressource Yol_Anticheat, le Watchdog intercepte l'événement et la relance instantanément.

Dépendances

  • cron (Système d'horloge ESX)
  • Yol_Anticheat

Fonctionnalités

  • Écoute en temps réel l'événement FiveM onResourceStop pour la ressource ciblée.
  • Planifie des vérifications cycliques via des tâches cron (toutes les minutes) au cas où l'anti-triche serait désactivé par des contournements complexes.
  • Envoie une alerte prioritaire avec mention de rôle sur votre Discord via la convar yol_ac_webhook.

Yol_Debugger

Système & Modération Dossier: [Others]/Yol_Debugger

Centralisateur de débogage. Il écoute les canaux système en continu pour détecter la moindre erreur fatale, crash ou exception de code, puis l'écrit dans un fichier log local et l'envoie sur Discord pour une correction immédiate.

Fonctionnalités

  • Console Listener : Filtre les messages de la console serveur à la recherche de mots-clés de crash (error, exception, traceback, failed).
  • NUI Exception Catcher : Intercepte les erreurs JavaScript silencieuses qui se produisent dans les interfaces UI des joueurs en jeu et les transmet au serveur.
  • Local logging : Écrit les rapports dans yol_errors.log (auto-limité à 300 lignes pour éviter la saturation disque).
  • Alerte instantanément les développeurs sur Discord via le webhook yol_ac_webhook.

Yol_Clothing

Créateur & Boutiques Dossier: Yol_Clothing

Système unifié et esthétique pour la gestion complète de l'apparence des joueurs : création initiale du personnage, magasins de vêtements, salons de coiffure, tatoueurs addon, et garde-robe.

Dépendances

  • oxmysql (Bdd)
  • ox_lib (UI/Menus)
  • screenshot-basic (requis pour générer les vignettes de tatouages)

Fonctionnalités

  • Cinématique de création : Caméra aérienne et panoramique, zoom dynamique en fonction de la zone éditée (visage, torse, jambes, pieds).
  • Tatouages Addon : Supporte plus de 210 tatouages personnalisés en plus des tatouages GTA vanilla (textures chargées dans le sous-dossier stream/).
  • Intégration ox_inventory : Synchronise les vêtements achetés avec le placard (wardrobe) du joueur en générant un item physique de tenue.
  • Auto-save : Enregistre l'état du joueur toutes les 30 secondes en base de données (colonne skin dans la table users).

Fichiers de Configuration & Commandes

  • shared/config.lua : Déclare les positions des boutiques, les tarifs par catégorie de vêtement, les tenues de départ des personnages et les décalages caméra.
  • Commande Admin : Taper /creator [playerId] ouvre de force l'éditeur de personnage pour la cible.
  • Commande Client : Taper /tenue dans la console F8 pour imprimer la configuration vestimentaire en format Lua.
  • Commande Admin de Tatouages : Taper /gentattoothumbs pour générer automatiquement les images miniatures des tatouages addon.

Yol_Core

Rôleplay & Immersion Dossier: [Others]/Yol_Core

Regroupe un grand nombre de fonctionnalités immersives, d'animations d'ambiance et d'interactions physiques pour enrichir l'expérience de jeu.

Dépendances

  • es_extended
  • ox_inventory
  • ox_target
  • pma-voice

Fonctionnalités

  • Interactions Ox_target : Permet de s'adosser à un mur, porter un joueur blessé ou consentant (/carry), s'asseoir ou s'allonger sur les bancs, chaises, canapés et lits de la map.
  • Survie et Hygiène : Utilisation des douches pour déstresser et se laver, toilettes, lavage de mains aux lavabos, consommation d'eau aux fontaines publiques. Système d'odeur (hygiène basse) faisant fuir les PNJ environnants.
  • Animation des Besoins : Plaintes physiques automatiques (se frotter le ventre, s'étouffer) lorsque la faim ou la soif descend sous la barre des 5%.
  • Indicateur vocal intelligent & Lip-sync : HUD affichant la portée de voix (Chuchoter, Normal, Crier). Lip-sync avancé qui calcule l'ouverture de la bouche du joueur en analysant le volume micro de pma-voice en temps réel.
  • Contrôle Minimap : La minimap (GPS) se désactive et n'apparaît à l'écran que si le joueur possède physiquement les items gps ou phone dans son inventaire.
  • Conteneurs Portables : Prise en charge des sacs à dos et boîtes de transport comme des extensions d'inventaire physiques (props portés visibles).

Fichiers de Configuration & Commandes

  • shared/config.lua : Active/désactive chaque sous-fonctionnalité, liste les props compatibles avec l'assise automatique, configure le rayon d'hygiène et les paliers d'inactivité (AFK).
  • Commande Admin : /menutarget ouvre le menu de placement des points interactifs (douches, lavabos, chaises fixes, etc.) enregistrés en BDD dans custom_props.json.
  • Touche N : Cycle entre les portées de voix (Chuchoter = 2m, Normal = 8m, Crier = 16m).

Yol_EMS

Métiers Dossier: [Others]/Yol_EMS

Script médical avancé intégrant un système de diagnostic visuel précis via un panel NUI représentant une silhouette humaine. Permet aux médecins de cibler précisément les blessures par zone corporelle et d'appliquer le traitement adéquat.

Dépendances

  • es_extended
  • ox_lib
  • ox_target
  • ox_inventory
  • oxmysql

Fonctionnalités

  • Panel Blessures NUI : Représentation anatomique interactive. Permet de consulter l'état des membres du patient (Tête, Torse, Bras gauche/droit, Jambe gauche/droit).
  • Localisation des dégâts : Les dégâts subis en jeu (tirs, chutes, collisions) sont localisés sur le membre affecté en fonction de la bone GTA touchée, avec des conséquences dynamiques (ex : boiter en cas de fracture à la jambe).
  • Matériel médical complet : Bandage (soigne éraflures), kit de suture (ferme coupures), attelle (répare fractures), garrot (stoppe saignement artériel), morphine/antidouleurs (gère le stress et la douleur), et le défibrillateur (réanime).

  • Touche F7 : Ouvre le Quick Menu EMS pour interagir avec un patient.
  • Commande Admin : Taper /clearinjuries [playerId] pour soigner instantanément toutes les blessures localisées d'un joueur.

Yol_Phone

Le smartphone ultime du serveur. Ce téléphone est entièrement indexé sur l'item d'inventaire physique. Si vous perdez, donnez ou vous faites voler votre téléphone, la carte SIM (numéro, messages, contacts, notes) suit l'item en temps réel.

  • oxmysql
  • xsound (obligatoire pour la musique et sonneries)

Fonctionnalités & Applications

  • Prop Téléphone Physique : Un modèle 3D de smartphone apparaît dans la main droite du joueur dès que l'interface est ouverte.
